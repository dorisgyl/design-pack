# design-pack

[English](README.md) · [简体中文](README.zh-CN.md)

A pack for the **NevoFlux browser**: a bundle of modern design **requirements / specs /
templates** that live in GBrain, plus a selection **dashboard** and two skills that retrieve
those design bases and generate finished UI (HTML) from them.

Inspired by [GoogleChrome/modern-web-guidance](https://github.com/GoogleChrome/modern-web-guidance)'s
"semantic search + Guide Fetch" idea — but retrieval is done by GBrain's built-in `query`
(hybrid vector + keyword), so there's no bundled local index.

## How it works

```
dashboard (pre-built, display only)          agent side
┌─────────────────────────────┐          ┌──────────────────────────────────────────┐
│ pick reqs/specs/templates +  │  agent.  │ design-build:                              │
│ a prompt → [design-pack:build]│  chat() │  get_page(selected slugs) + query(expand)  │
└─────────────────────────────┘ ───────▶ │  → create_artifact() new canvas            │
                                          └──────────────────────────────────────────┘
```

- **Canvas can't call GBrain directly** (MCP tools aren't directly callable) → retrieval and
  generation both happen on the agent side.
- **The iframe blocks external URLs** → the dashboard is fully self-contained; sample images are
  inlined as base64.
- Content lives in GBrain under `packs/design-pack/...` and can be extended with `design-curate`.

## Layout

```
pack.toml                                  # manifest (namespace = packs/design-pack)
components/
  skills/
    design-build/SKILL.md                  # retrieve + generate (shared by dashboard & sidebar)
    design-curate/SKILL.md                 # import/extend + rebuild the dashboard
  seed/{requirements,specs,templates}/*.md # original design content (seeded only-if-absent)
  seed/templates/<id>.md + <id>-zh.md      # imported from html-anything (en+zh), content → NevoFlux
  seed/specs/{taste,workflow}/*.md         # imported from taste-skill: 9 taste + 4 workflow
  seed/guides/<category>/*.md              # 137 guides imported from modern-web-guidance
  canvas-app/dist/index.html               # pre-built dashboard (generated by the build script)
scripts/
  frontmatter.mjs                          # tiny dependency-free frontmatter parser
  import-guides.mjs                        # one-shot: modern-web-guidance/guides → seed/guides/**
  import-taste.mjs                         # one-shot: taste-skill/skills → seed/specs/**
  sync-pack-seeds.mjs                      # regenerate pack.toml's [[components.seed]] from seed/**
  build-dashboard.mjs                      # seeds → dist/index.html (reference for curate rebuild)
  validate-pack.mjs                        # invariant checks (local validation gate)
```

## Content sources

- **Original design content (8)**: accessibility/responsive baselines, color/type/spacing specs,
  3 UI templates.
- **Imported guides (137)**: from [modern-web-guidance](https://github.com/GoogleChrome/modern-web-guidance)'s
  `guides/`, classified as `type: spec`, by category (accessibility / css / forms / performance /
  user-experience / …) under `packs/design-pack/guides/<category>/`. Grouped in the dashboard as
  "指南 · <category>", with a filter. Re-import: `node scripts/import-guides.mjs [guides-dir]`.
- **Imported taste-skill (13)**: from `taste-skill/skills`, all `type: spec`, in two collections:
  - `collection: taste` (9 → `specs/taste/`): brandkit, brutalist, minimalist, soft, taste (+v1),
    redesign, stitch, gpt-taste — aesthetic/taste systems. Dashboard group "设计品味 (taste)".
  - `collection: workflow` (4 → `specs/workflow/`): output (anti-truncation), image-to-code,
    imagegen-web, imagegen-mobile — output/asset-generation process. Dashboard group
    "工作流 / 产出 (workflow)".
  - Re-import: `node scripts/import-taste.mjs [skills-dir]` (the classification lives in the
    script's `WORKFLOW` set).
- **Imported templates (from html-anything)**: each `SKILL.md` + `example.html` becomes a
  `type: template` page under `packs/design-pack/templates/`, in **two language versions**
  (`<id>` en + `<id>-zh` zh) for bilingual GBrain retrieval. The example HTML keeps its original
  format/style; only the visible content is rewritten to be about NevoFlux. `example.md` (sample
  domain data) is skipped. Dashboard groups templates as "模板 · <category>"; zh pages are
  retrieval-only (one card per template). Adapted via a multi-agent workflow.

## Content model

Each requirement/spec/template = one GBrain page (markdown + frontmatter):
`slug` · `type` (requirement|spec|template) · `title` · `description` · `tags` ·
`sample_image` (templates only). The dashboard uses only the lightweight catalog
(title/description/thumbnail); bodies stay in GBrain and are retrieved by the agent.

## Develop

```bash
node scripts/import-guides.mjs      # (optional) re-import modern-web-guidance guides
node scripts/import-taste.mjs       # (optional) re-import taste-skill skills
node scripts/sync-pack-seeds.mjs    # after adding/removing seeds, sync pack.toml's seed list
node scripts/build-dashboard.mjs    # after editing seeds, rebuild the dashboard
node scripts/validate-pack.mjs      # invariant checks (Seed⊆Protected, frontmatter, self-contained…)
```

## Install (on a machine running the NevoFlux daemon)

```bash
nevoflux pack validate pack.toml    # platform-level validation (CLI not installed in this repo's env)
nevoflux pack install pack.toml
```

Sandbox trial: point `XDG_CONFIG_HOME` and `NEVOFLUX_DATA_DIR` at a temp dir and do
validate → install → uninstall.

## Tool names (verified against source)

Cross-checked against `nevoflux-agent/crates/daemon/src/resources/gbrain-tools.json` and
NevoFlux's own skills (`brain-recall` / `brain-capture` / `app`):

- **GBrain tool names are correct**: `get_page` / `put_page` / `query` / `search` / `list_pages` /
  `file_upload` (params `path`, `page_slug`) / `file_url` (param `storage_path`).
- **Access pattern**: skills do NOT list these tool names in `allowed_tools`; they declare
  `tool_search` + `tool_call_dynamic` (discover + invoke dynamically). Both skills here follow that.
- **The frontmatter key is `allowed_tools` (underscore)**, not the `allowed-tools` shown in the
  pack docs example.
- **Canvas tools**: `create_artifact` (new; ambient — the `app` skill uses it with
  `allowed_tools: []`), `browser_read_artifact` / `browser_edit_artifact` (read/edit an existing
  artifact).
